## Overview

**TransAsia-222-Interactive-Case-Analysis-Simulator** is a standalone interactive HTML dashboard designed to help users examine how **environmental pressure**, **crew decisions**, and **procedural breakdown** evolved into the final aviation mishap outcome.

Rather than presenting the event as a simple timeline alone, the simulator organizes the accident into multiple visual layers:

- an **influence network** showing category-theme relationships
- a **warning monitor** surfacing risk signals
- a **timeline feed** showing event progression
- a **seat-specific outcome image** interpreting survivability patterns
- a **report summary** linking the visual model to the documented mishap narrative

This makes the project useful as both a **portfolio piece** and a **teaching / briefing tool** for aviation safety, human factors, and accident analysis.

---

## Features

| Feature | Description | Why it matters |
|---|---|---|
| **Influence Network** | Visualizes dominant category-theme connections in the mishap model | Helps users identify the strongest conceptual drivers in the event sequence |
| **Interactive Filters** | Theme, category, and minimum-weight controls refine the network view | Makes the dashboard easier to explore and compare |
| **Warning Monitor** | Highlights risk signals and unsafe conditions as they emerge | Supports fast interpretation of threat buildup |
| **Timeline Feed** | Shows how the mishap unfolds across key moments | Connects abstract themes to event progression |
| **Seat-Specific Outcome Image** | Interprets survivability as spatially uneven across the cabin | Links final outcomes to impact, breakup, fire, and egress conditions |
| **Report Summary** | Provides narrative grounding from the accident interpretation | Bridges the visuals with documented findings |
| **Standalone HTML Delivery** | Runs directly in a browser with minimal setup | Easy to share, demo, archive, and include in a portfolio |

## Demo Workflow

To use the simulator, begin with the **influence network** on the left to identify dominant category-theme connections, then adjust the **theme**, **category**, and **minimum-weight** controls to refine what is displayed.

Next, review the **warning monitor** and **timeline feed** on the right to track how risk signals and event progression align with the filtered network.

Finally, use the **seat-specific outcome image** and **report summary** below to connect the visual analytics to the aviation mishap.
